# 时间线滚动条组件需求说明

## 组件概述
底部显示的时间线进度条，用于选择历史时间点。

## 核心需求

### 1. 时间范围
- **最左边**：默认当前时间向前 20 年（可通过 `TimelineState(pastYears:)` 配置）
- **最右边**：当前时间（实时）

### 2. 刻度显示
- 每年显示12个刻度，代表12个月份
- 年份标签显示在对应1月刻度的**上方**（不是左侧）
- 刻度样式区分：
  - 1月刻度最高最粗
  - 季度月份（4月、7月、10月）中等
  - 其他月份最小

### 3. 初始状态
- **中轴默认对准当前日期**
- 初始化时滚动条应该滚动到最右侧，让当前日期的刻度对准中轴指示器

### 4. 滚动交互
- 支持左右自由滚动
- **向左滚动**：查看过去的时间（默认可滚动到当前时间向前 20 年）
- **向右滚动**：查看更近的时间（最多滚动到当前时间）
- 滚动后**不应自动跳回**，用户滚动到哪里就停在哪里

### 5. 选择机制
- 当刻度滚动到**中轴位置**时，认为选中了该时间点
- 中轴有固定的指示器（视觉提示）
- 选中的时间应同步到 `TimelineState.selectedDate`

### 6. 状态管理
- 使用 `@Observable` 类型的 `TimelineState` 管理状态
- 父组件通过传入 `TimelineState` 实例与时间线同步状态
- 父组件可以读取 `selectedDate` 获取当前选中的日期

## 视觉效果
- 整体使用 `.glassEffect()` 液态玻璃效果
- 中轴指示器固定在中央，不随滚动移动
- 高度固定 44pt

## 技术实现
- 使用 `ScrollViewReader` + `task` 实现初始滚动到当前日期
- 使用 `onScrollGeometryChange` 监听滚动位置更新 `selectedDate`
- 延迟 100ms 确保布局完成后再执行初始滚动
